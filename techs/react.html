React

My main JavaScript UI library — components, hooks, and state where it matters.

I started using React because its component model just made sense — reusable, dynamic, and easier to scale than plain HTML. I focus on small, composable pieces and predictable data-fetching. Accessibility and keyboard flows are part of every UI decision I make.

  • Clean component boundaries, thoughtful state
  • Component reusability
  • Initial load might be slow, but blazing fast after that

React helps me move fast without sacrificing structure or clarity.
